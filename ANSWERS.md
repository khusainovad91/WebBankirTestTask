#### Вопрос 1

В `BaseTest` используется `implicitlyWait(10s)`. Ваш коллега добавил явный `WebDriverWait(driver, 15s)` в одном из тестов. При запуске тесты стали нестабильными - иногда падают, иногда нет, без видимой причины. Объясните возможную механику происходящего и как бы вы исправили проблему.

#### Ответ 
Очевидно, что явное ожидание превышает общее неявное неявное. Решение: создать переменную BaseTest с параметром неявного ожидания и сетапить ее в таких длинных кейсах в @Before

#### Вопрос 2

Тест-сьют из 200 UI тестов стабильно проходит локально, но в CI падает ~15% тестов с разными ошибками: `StaleElementReferenceException`, `NoSuchElementException`, и иногда - неожиданный `401` от API. Тесты перезапускаются и проходят. Какие 3 наиболее вероятные причины нестабильности вы бы проверили в первую очередь, и как бы вы их диагностировали?

#### Ответ
Причины:
1) Тачка в CI работает медленнее локальной, страницы не успевают отрисовываться
2) По 401 скорее всего протухает токен для авторизации. Тестов много, скорее всего используют одну аутентифкацию в @BeforeAll
3) Проверить кейсы на использование Thread.sleep

Диагностику бы проводил по логам или по отчету allure. Возможно необходимо будет расширить логирование

#### Вопрос 3

```java
public class TestDeleteUser {
	@Test
	void testDeleteUser() {
	    String userId = "user-123";
	    apiClient.deleteUser(userId);
	
	    Response response = apiClient.getUserById(userId);
	    assertThat(response.statusCode()).isEqualTo(404);
	}
}
```
#### Ответ
+ Отсутствует проверка на удаление
Response responseDelete = apiClient.deleteUser(userId);
assertThat(responseDelete.statusCode()).isEqualTo(200);
+ не факт, что такое пользователь существует, в прекондишнс стоило бы его сперва создать
